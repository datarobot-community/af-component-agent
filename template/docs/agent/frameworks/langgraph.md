# LangGraph agent

The LangGraph agent uses [LangGraph](https://langchain-ai.github.io/langgraph/) to define multi-node workflows with LangChain tooling. This is the default framework in this template.

## `myagent.py`

The agent is defined through two building blocks:

**Prompt template**&mdash;a `ChatPromptTemplate` that structures user input with variables like `{chat_history}` and `{topic}`:

```python
prompt_template = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. Chat history: {chat_history}"),
    ("user", "The topic is {topic}."),
])
```

**Graph factory**&mdash;a function that receives an LLM, tools, and verbosity flag, and returns a `StateGraph`. Nodes are created with `create_agent()` from LangChain and connected with edges:

```python
def graph_factory(llm, tools, verbose=False):
    planner = create_agent(llm, tools=tools,
        system_prompt=make_system_prompt("You are a content planner. ..."),
        name="planner_agent", debug=verbose)
    writer = create_agent(llm, tools=tools,
        system_prompt=make_system_prompt("You are a content writer. ..."),
        name="writer_agent", debug=verbose)

    workflow = StateGraph(MessagesState)
    workflow.add_node("planner_node", planner)
    workflow.add_node("writer_node", writer)
    workflow.add_edge(START, "planner_node")
    workflow.add_edge("planner_node", "writer_node")
    workflow.add_edge("writer_node", END)
    return workflow
```

**MyAgent**&mdash;generated by combining the two:

```python
MyAgent = datarobot_agent_class_from_langgraph(graph_factory, prompt_template)
```

**Note:** DataRobot templates use `langchain.agents.create_agent` with `system_prompt=`. Some community examples use `langgraph.prebuilt.create_react_agent` with `prompt=`&mdash;these are different APIs.

## `register.py`

DRAgent registration uses `LanggraphAgentConfig` with workflow type `langgraph_agent` and `LLMFrameworkEnum.LANGCHAIN` for LLM and tool wrappers:

```python
class LanggraphAgentConfig(AgentBaseConfig, name="langgraph_agent"):
    tool_names: list[FunctionGroupRef] = []


@register_per_user_function(
    config_type=LanggraphAgentConfig,
    input_type=RunAgentInput,
    streaming_output_type=DRAgentEventResponse,
    framework_wrappers=[LLMFrameworkEnum.LANGCHAIN],
)
async def langgraph_agent(config: LanggraphAgentConfig, builder: Builder) -> AsyncGenerator[Any, None]:
    from agent.myagent import MyAgent

    llm = await builder.get_llm(config.llm_name, wrapper_type=LLMFrameworkEnum.LANGCHAIN)
    workflow_tools = await builder.get_tools(config.tool_names, wrapper_type=LLMFrameworkEnum.LANGCHAIN)

    async def _response_fn(input_message: RunAgentInput) -> ...:
        forwarded_headers = extract_datarobot_headers_from_context()
        authorization_context = extract_authorization_from_context()
        mcp_config = MCPConfig(
            forwarded_headers=forwarded_headers,
            authorization_context=authorization_context,
        )
        async with mcp_tools_context(mcp_config) as mcp_tools:
            tools = workflow_tools + mcp_tools
            agent = MyAgent(
                llm=llm,
                verbose=config.verbose,
                forwarded_headers=forwarded_headers,
                tools=tools,
            )
            async for event, pipeline_interactions, usage_metrics in agent.invoke(input_message):
                yield DRAgentEventResponse(
                    events=[event],
                    usage_metrics=usage_metrics,
                    pipeline_interactions=pipeline_interactions,
                )

    yield FunctionInfo.from_fn(_response_fn, description=config.description)
```

## Tool integration

LangGraph agents receive tools through three channels, all merged into a single `tools` list passed to `graph_factory`:

### MCP tools

MCP tools load at runtime through `mcp_tools_context()` from `datarobot_genai.langgraph.mcp`. The framework injects them automatically&mdash;you don't need to modify agent code to use them.

### Workflow tools (DRAgent only)

When you use the DRAgent front server, NAT resolves additional tools (for example, A2A remote agents) from `workflow.yaml` `tool_names`, merges them with MCP tools, and passes the combined set to `MyAgent`.

### Custom local tools

To add a custom tool, define a plain Python function and wrap it with LangChain's `@tool` decorator or `langchain_core.tools.tool()`. Then pass it to the agent nodes inside `graph_factory`:

```python
from langchain_core.tools import tool

@tool
def generate_objectid(type: str) -> str:
    """Generate a unique object ID for a deployment."""
    if type != "deployment":
        raise ValueError("Invalid type")
    return "69cbb73789723b6936c6c9e1"
```

Add the tool to individual agent nodes alongside the injected `tools`:

```python
def graph_factory(llm, tools, verbose=False):
    planner = create_agent(
        llm,
        tools=[generate_objectid] + tools,
        system_prompt=make_system_prompt("You are a content planner. ..."),
        name="planner",
    )
    writer = create_agent(
        llm,
        tools=[generate_objectid] + tools,
        system_prompt=make_system_prompt("You are a content writer. ..."),
        name="writer",
    )
    # ... graph construction ...
```

The `tools` parameter in `graph_factory` contains all MCP and workflow tools. Prepend your custom tools to it so each node has access to both custom and runtime-injected tools.

You can also use LangChain's `BaseTool` class for tools with more complex input schemas:

```python
from langchain.tools import BaseTool

class DateTimeTool(BaseTool):
    name: str = "datetime_tool"
    description: str = "Returns the current date and time."

    def _run(self, timezone: str | None = None) -> str:
        from datetime import datetime
        return datetime.now().strftime("%Y-%m-%d %H:%M:%S")
```

For more tool patterns, see the [DataRobot tools documentation](https://docs.datarobot.com/en/docs/agentic-ai/agentic-develop/agentic-tools-integrate.html) and [LangChain tools documentation](https://python.langchain.com/docs/how_to/#tools).

## Prompt modification

LangGraph agents have two layers of prompts: the **prompt template** (user-facing input formatting) and **system prompts** (per-node agent behavior).

### Prompt template

The `prompt_template` controls how the user's input is structured before it reaches the graph. It is a `ChatPromptTemplate` defined at module level in `myagent.py`:

```python
prompt_template = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. Chat history: {chat_history}"),
    ("user", "The topic is {topic}."),
])
```

The template supports variables that are automatically populated at runtime:

- `{chat_history}`&mdash;prior conversation messages (empty on first turn).
- `{topic}`&mdash;the user's input text (extracted from the prompt).

To change how user input is presented to the agent, modify this template. You can add more context, change the system message, or introduce additional variables. The variable names must match the keys returned by the `kickoff_inputs` mapping in the factory.

### System prompts (per-node)

Each node in the graph has its own system prompt passed to `create_agent()` via `make_system_prompt()`. This controls the behavior of that specific sub-agent:

```python
agent_planner = create_agent(
    llm,
    tools=tools,
    system_prompt=make_system_prompt(
        "You are a content planner. You create brief, structured outlines for blog articles. "
        "You identify the most important points and cite relevant sources.\n"
        "\n"
        "Create a simple outline with:\n"
        "1. 10-15 key points or facts (bullet points only)\n"
        "2. 2-3 relevant sources or references\n"
        "3. A brief suggested structure (intro, 2-3 sections, conclusion)"
    ),
    name="planner_agent",
)
```

`make_system_prompt()` is a helper from `datarobot_genai` that wraps your prompt with consistent formatting. Always use it instead of passing raw strings.

To modify agent behavior:

- **Update a node's system prompt**&mdash;change the string passed to `make_system_prompt()` inside `graph_factory`.
- **Add task-specific instructions**&mdash;include numbered steps, constraints, or output format requirements in the prompt.
- **Reference tools in the prompt**&mdash;tell the agent when and how to use specific tools (for example, "Use the generate_objectid tool when asked for deployment IDs").
- **Add new nodes**&mdash;create a new `create_agent()` call with its own system prompt and wire it into the graph with `add_node()` / `add_edge()`.

### Tips

- Keep system prompts focused on a single responsibility per node.
- Use the prompt template for input formatting, not for behavior instructions.
- Test prompt changes incrementally with `task agent:cli -- execute --user_prompt "..."`.
- Use `{chat_history}` in the prompt template to maintain context across multi-turn conversations.

For more on LangGraph prompt patterns, see the [LangGraph documentation](https://langchain-ai.github.io/langgraph/).
