# CrewAI agent migration to 11.8.8

This guide covers migrating a CrewAI agent from the class-based layout (pre-11.8.8) to the new factory-based layout introduced in [af-component-agent#474](https://github.com/datarobot-community/af-component-agent/pull/474).

## Summary of changes

- `MyAgent` is no longer a hand-written class extending `CrewAIAgent`. It is generated by `datarobot_agent_class_from_crew()`.
- The `llm()` method is removed. LLM creation is decoupled from the agent and handled by `get_llm()` from `datarobot_genai.crewai.llm`.
- The `Config` class import is removed. Configuration is handled internally by `get_llm()`.
- Agents, tasks, crew, and kickoff inputs are defined at module level instead of as class properties/methods.

## Migration steps

### 1. Update imports

**Before:**

```python
from crewai import LLM, Agent, Crew, Process, Task
from crewai.tools import BaseTool
from datarobot_genai.crewai.agent import CrewAIAgent

from agent.config import Config
```

**After:**

```python
from crewai import Agent, Crew, Process, Task
from datarobot_genai.crewai.agent import datarobot_agent_class_from_crew
from datarobot_genai.crewai.llm import get_llm
```

Remove imports of `LLM`, `BaseTool` (if only used for type hints in `__init__`), `CrewAIAgent`, and `Config`.

### 2. Move agents to module level

**Before:**

```python
class MyAgent(CrewAIAgent):
    @property
    def agent_planner(self) -> Agent:
        return Agent(
            role="Planner",
            goal="Create a simple, focused outline for {topic}.",
            backstory="You create brief, structured outlines...",
            allow_delegation=False,
            verbose=self.verbose,
            llm=self.llm(),
            tools=self.tools,
        )
```

**After:**

```python
llm = get_llm()

agent_planner = Agent(
    role="Planner",
    goal="Create a simple, focused outline for {topic}.",
    backstory=make_system_prompt("You create brief, structured outlines..."),
    allow_delegation=False,
    verbose=True,
    llm=llm,
)
```

Key differences:
- `self.llm()` &rarr; `llm` (module-level, created by `get_llm()`)
- `self.tools` &rarr; removed from agent definition (tools are injected at runtime)
- `self.verbose` &rarr; hardcoded `True` (or your preferred default)
- `backstory` now uses `make_system_prompt()` for consistent formatting

### 3. Move tasks to module level

**Before:**

```python
class MyAgent(CrewAIAgent):
    @property
    def task_plan(self) -> Task:
        return Task(
            description="Create a simple outline for {topic} with: ...",
            expected_output="A simple outline...",
            agent=self.agent_planner,
        )
```

**After:**

```python
task_plan = Task(
    description="Create a simple outline for {topic} with: ...",
    expected_output="A simple outline...",
    agent=agent_planner,
)
```

### 4. Move crew to module level

**Before:**

```python
class MyAgent(CrewAIAgent):
    def crew(self) -> Crew:
        return Crew(
            agents=self.agents,
            tasks=self.tasks,
            verbose=self.verbose,
            process=Process.sequential,
            stream=self.stream,
        )
```

**After:**

```python
crew = Crew(
    agents=agents,
    tasks=tasks,
    verbose=True,
    process=Process.sequential,
    stream=False,
)
```

Note: `stream` is set to `False` at module level. In DRAgent mode, `register.py` sets `crew.stream = True` at runtime.

### 5. Define `kickoff_inputs` at module level

**Before:**

```python
class MyAgent(CrewAIAgent):
    def make_kickoff_inputs(self, user_prompt_content: str) -> dict[str, Any]:
        return {
            "topic": str(user_prompt_content),
            "chat_history": "",
        }
```

**After:**

```python
kickoff_inputs = lambda user_prompt_content: {
    "topic": str(user_prompt_content),
    "chat_history": "",
}
```

### 6. Generate `MyAgent` with the factory

Replace the entire `MyAgent` class with:

```python
MyAgent = datarobot_agent_class_from_crew(crew, agents, tasks, kickoff_inputs)
```

### 7. Delete the `__init__` and `llm()` methods

The entire `__init__` method and the `llm()` method are no longer needed. Remove them along with any `self.config`, `self.default_model`, `self._llm` references.

### 8. Update `custompy_adaptor`

**Before:**

```python
async def custompy_adaptor(completion_create_params, ...):
    ...
    agent = MyAgent(
        model=completion_create_params.get("model"),
        verbose=...,
        timeout=...,
        forwarded_headers=...,
        stream=False,
    )
```

**After:**

```python
_PLACEHOLDER_MODELS = frozenset({"unknown"})

async def custompy_adaptor(completion_create_params, ...):
    ...
    model_name = completion_create_params.get("model")
    agent = MyAgent(
        llm=get_llm(
            model_name=model_name if model_name not in _PLACEHOLDER_MODELS else None,
            parameters={"stream_options": None},
        ),
        verbose=...,
        timeout=...,
        forwarded_headers=...,
    )
```

Key differences:
- `model=` &rarr; `llm=get_llm(model_name=...)`.
- `stream=False` parameter is removed (controlled at the crew level).
- `parameters={"stream_options": None}` is passed to `get_llm()` for CrewAI compatibility.

### 9. Update tests

- Remove tests for `llm()` and `__init__` parameters.
- Replace `MyAgent(model=..., api_key=...)` with `MyAgent(llm=Mock(), ...)`.
- Test module-level agents, tasks, and crew directly.
- Test `custompy_adaptor` with mock `get_llm` to verify placeholder model filtering.
