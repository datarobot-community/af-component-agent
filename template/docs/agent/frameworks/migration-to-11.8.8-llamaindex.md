# LlamaIndex agent migration to 11.8.8

This guide covers migrating a LlamaIndex agent from the class-based layout (pre-11.8.8) to the new factory-based layout introduced in [af-component-agent#474](https://github.com/datarobot-community/af-component-agent/pull/474).

## Summary of changes

- `MyAgent` is no longer a hand-written class extending `LlamaIndexAgent`. It is generated by `datarobot_agent_class_from_llamaindex()`.
- The `llm()` method is removed. LLM creation is decoupled from the agent and handled by `get_llm()` from `datarobot_genai.llama_index.llm`.
- The `Config` class import is removed. Configuration is handled internally by `get_llm()`.
- Agents, workflow, state tools, and `extract_response_text` are defined at module level instead of as class properties/methods.
- `make_input_message()` is no longer needed&mdash;the factory handles input formatting.

## Migration steps

### 1. Update imports

**Before:**

```python
from datarobot_genai.llama_index import DataRobotLiteLLM
from datarobot_genai.llama_index.agent import LlamaIndexAgent
from llama_index.core.llms import LLM as BaseLLM
from llama_index.core.tools import BaseTool

from agent.config import Config
```

**After:**

```python
from datarobot_genai.llama_index.agent import datarobot_agent_class_from_llamaindex
from datarobot_genai.llama_index.llm import get_llm
from llama_index.core.agent.workflow import BaseWorkflowAgent
from llama_index.llms.litellm import LiteLLM
```

Remove imports of `DataRobotLiteLLM`, `LlamaIndexAgent`, `BaseLLM`, and `Config`.

### 2. Define a module-level LLM placeholder

The module-level LLM is a placeholder that gets replaced at runtime when `custompy_adaptor` or DRAgent injects the real LLM:

```python
llm = LiteLLM(model="placeholder")
```

### 3. Move state tools to module level

**Before:**

```python
class MyAgent(LlamaIndexAgent):
    @staticmethod
    async def planner_notes_tool(ctx: Context, notes: str, notes_title: str) -> str:
        async with ctx.store.edit_state() as current_state:
            if "planner_notes" not in current_state:
                current_state["planner_notes"] = {}
            current_state["planner_notes"][notes_title] = notes
        return "Notes recorded."
```

**After:**

```python
async def planner_notes_tool(ctx: Context, notes: str, notes_title: str) -> str:
    """Record planning notes into the shared workflow state."""
    async with ctx.store.edit_state() as current_state:
        if "planner_notes" not in current_state:
            current_state["planner_notes"] = {}
        current_state["planner_notes"][notes_title] = notes
    return "Notes recorded."
```

These are now plain module-level async functions.

### 4. Move agents to module level

**Before:**

```python
class MyAgent(LlamaIndexAgent):
    @property
    def agent_planner(self) -> FunctionAgent:
        return FunctionAgent(
            name="planner_agent",
            description="Useful for planning content...",
            system_prompt="You are the planner_agent...",
            llm=self.llm(),
            tools=[self.planner_notes_tool, *self.tools],
            can_handoff_to=["writer_agent"],
        )
```

**After:**

```python
agent_planner = FunctionAgent(
    name="planner_agent",
    description="Useful for planning content...",
    system_prompt=make_system_prompt("You are the planner_agent..."),
    llm=llm,
    tools=[planner_notes_tool],
    can_handoff_to=["writer_agent"],
)
```

Key differences:
- `self.llm()` &rarr; `llm` (module-level placeholder)
- `[self.planner_notes_tool, *self.tools]` &rarr; `[planner_notes_tool]` (MCP/runtime tools are injected separately)
- `system_prompt` now uses `make_system_prompt()` for consistent formatting

### 5. Move workflow to module level

**Before:**

```python
class MyAgent(LlamaIndexAgent):
    async def build_workflow(self) -> AgentWorkflow:
        return AgentWorkflow(
            agents=[self.agent_planner, self.agent_writer],
            root_agent=self.agent_planner.name,
            initial_state={"planner_notes": {}, "report_content": "Not written yet."},
        )
```

**After:**

```python
agents: list[BaseWorkflowAgent] = [agent_planner, agent_writer]

workflow = AgentWorkflow(
    agents=agents,
    root_agent=agent_planner.name,
    initial_state={"planner_notes": {}, "report_content": "Not written yet."},
)
```

### 6. Move `extract_response_text` to module level

**Before:**

```python
class MyAgent(LlamaIndexAgent):
    def extract_response_text(self, result_state: Any, events: list[Any]) -> str:
        if isinstance(result_state, dict) and "report_content" in result_state:
            return str(result_state["report_content"])
        ...
```

**After:**

```python
def extract_response_text(result_state: Any, events: list[Any]) -> str:
    """Safely extract the final report text from the workflow state or events."""
    if isinstance(result_state, dict) and "report_content" in result_state:
        return str(result_state["report_content"])
    ...
```

Note: `self` is removed from the signature.

### 7. Remove `make_input_message`

The `make_input_message()` method is no longer needed. The factory handles input formatting internally.

### 8. Generate `MyAgent` with the factory

Replace the entire `MyAgent` class with:

```python
MyAgent = datarobot_agent_class_from_llamaindex(workflow, agents, extract_response_text)
```

### 9. Delete the `__init__` and `llm()` methods

The entire `__init__` method and the `llm()` method are no longer needed. Remove them along with any `self.config`, `self.default_model`, `self._llm` references.

### 10. Update `custompy_adaptor`

**Before:**

```python
async def custompy_adaptor(completion_create_params, ...):
    ...
    agent = MyAgent(
        model=completion_create_params.get("model"),
        verbose=...,
        timeout=...,
        forwarded_headers=...,
    )
```

**After:**

```python
_PLACEHOLDER_MODELS = frozenset({"unknown"})

async def custompy_adaptor(completion_create_params, ...):
    ...
    model_name = completion_create_params.get("model")
    agent = MyAgent(
        llm=get_llm(
            model_name=model_name if model_name not in _PLACEHOLDER_MODELS else None
        ),
        verbose=...,
        timeout=...,
        forwarded_headers=...,
    )
```

### 11. Update tests

- Remove tests for `llm()`, `__init__`, and `make_input_message`.
- Replace `MyAgent(model=..., api_key=...)` with `MyAgent(llm=Mock(), ...)`.
- Test module-level agents, workflow, and `extract_response_text` directly.
- Test `custompy_adaptor` with mock `get_llm` to verify placeholder model filtering.
